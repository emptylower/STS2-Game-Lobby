# STS2 LAN Connect 使用说明

## 进入大厅

1. 启动游戏并进入多人首页
2. 点击 `游戏大厅`
3. 直接在大厅里完成建房、刷新和加入
4. 如果默认大厅拥堵或不可用，可先点标题栏的 `切换服务器`

## 顶部公告

- 大厅顶部会显示当前服务器下发的公告轮播
- 公告支持 `更新`、`活动`、`警告`、`信息` 四类视觉样式
- 公告条使用左侧更亮、右侧更暗的暖色渐变背景，不做滚动文字动画
- 桌面端可用左右箭头和右侧点状页码切换
- 紧凑横屏模式会隐藏箭头，并改为 `1/N` 数字指示
- 公告默认每 6 秒自动切换，鼠标悬停时会暂停，底部进度条会从左往右累积

## 大厅列表操作

- 房间列表支持关键词搜索、分页和可叠加筛选
- 筛选支持 `公开`、`上锁`、`可加入`
- `公开` / `上锁` 互斥，再次点击当前筛选可取消
- `可加入` 会过滤掉当前不可加入的房间
- 房间列表支持鼠标滚轮滚动
- 移动端可按住列表区域上下滑动
- 单击房间卡片会选中目标房间
- 双击房间卡片会直接尝试加入
- 房间卡片会显示状态、真实游戏版本、真实 MOD 版本和 `relay` 就绪状态
- 如果连续刷新失败，顶部状态条会提示建议切服

## 房主流程

1. 打开 `游戏大厅`
2. 点击 `创建房间`
3. 填写房间名，选择房间类型和最大人数（4-8 人），可选填写密码
4. 发布成功后，客户端会自动：
   - 启动本地 ENet Host
   - 向大厅服务注册房间
   - 持续发送心跳保活

## 玩家流程

1. 打开 `游戏大厅`
2. 点击 `刷新大厅`
3. 用搜索和筛选定位目标房间
4. 如果刷新失败或延迟异常，可先用 `切换服务器` 换一个大厅
5. 选择目标房间并加入
6. 如果房间有密码，按提示输入密码

## 房间聊天

- 进入已连接房间后，右上角会出现 `房间聊天` 按钮
- 点击后可展开聊天面板，输入内容后按 `Enter` 或点 `发送`
- 面板收起时，如果有其他玩家发来新消息，会显示未读角标
- 标题栏和按钮支持长按拖动，松手后会把位置保存到本地配置
- 聊天走大厅控制通道，只在当前房间内广播，不会写进续局存档

## 多人续局

- 房主重新进入一个已存在的多人续局存档时，当前续局会自动重新发布到大厅
- 这个绑定会沿用该续局存档对应的房间信息，而不是要求重新手动建房
- 如果该续局还有多个空闲角色槽位，加入方会先看到角色选择，再进入联机
- 角色选择弹窗会同时显示角色名和原玩家名（如"铁甲战士（小明）"），方便多人选同角色时准确找回自己的槽位

## 加入等待提示

- 点击加入房间后，会出现加载中的提示弹窗
- 弹窗会按阶段更新，例如：
  - 正在向大厅申请加入
  - 大厅已响应，开始连房主
  - 正在尝试公网 / 局域网候选地址
  - 直连超时后自动尝试 relay fallback
  - 连接成功，进入联机界面
- 如果加入失败，弹窗会尽量区分版本不一致、MOD 不一致、房间已开局、房间已满或具体网络失败原因

## 调试报告

- 大厅设置区提供 `复制本地调试报告`
- 报告会包含当前选中房间的 `roomId`、本地平台玩家 ID、存档快照和最近相关客户端日志
- 如果你要向开发者反馈问题，优先提供这份报告

## 网络说明

- 当前大厅的默认连接策略由安装包内的 `lobby-defaults.json` 决定，可能是 `direct-first`、`relay-first` 或 `relay-only`
- 当前公开包默认使用阿里云大厅 `47.111.146.69:8787`，公共服务器目录是 `47.111.146.69:18787`，并固定为 `test_relaxed + relay-only`
- MOD 内置 5-8 人支持，建房时可选最大人数，自动适配难度缩放、营地座位、商店布局和宝箱分配
- 如果检测到 RMP 等外部扩展人数 MOD，内置补丁会自动跳过以避免冲突
- 当前版本会自动把房间人数元数据同步到大厅服务
- `切换服务器` 会从中心服务器拉取可用大厅列表，并把你选择的大厅写入当前客户端覆盖设置
- 开发网络设置里可单独填写 `中心服务器覆盖`，用于替换公共大厅列表来源
- 大厅显示的服务延迟来自独立探测，不是房间列表接口总耗时
- 房主机器如果能让 `33771/UDP` 直连可达，加入速度会更快
- 服务端如果启用了 relay fallback，需要放行 `39000-39149/UDP`
- `WS /control` 主要承担大厅协调、房主会话保活和房间聊天，不替代游戏联机数据通路

## 设置说明

- 普通玩家通常只需要填写 `玩家名`
- 普通玩家切环境时，优先使用 `切换服务器`
- 只有排障或切换环境时，才需要展开 `开发网络设置`
- 如果你要切换公共服务器目录来源，可在开发网络设置里改 `中心服务器覆盖`
- 如果安装包已经附带 `lobby-defaults.json`，默认大厅地址会自动生效，但不会默认明文展示在 UI 中
- 当前发布包的 MOD 版本号以 `mods/sts2_lan_connect/sts2_lan_connect.json` 为准

## 常见问题

### 大厅里看不到房间

- 先确认大厅服务健康检查正常
- 再确认搜索关键词、分页和筛选没有把目标房间漏掉
- 再尝试点击 `切换服务器`，切到另一个可用大厅后重新刷新
- 再确认房主房间是否发布成功

### 大厅能刷出来，但加入总是超时

- `创建房间`、`刷新大厅`、`加入申请` 这些控制面请求走的是 `HTTP/TCP`
- 真正的直连和 relay fallback 走的是 `UDP`
- 可先尝试 `切换服务器`，排除单个大厅节点拥堵或抖动
- 如果你启用了 `Clash`、`Surge`、`TUN`、系统全局代理或本地网络过滤工具，必须让大厅服务器 IP 走 `DIRECT`

### 提示 MOD 不一致

- 所有联机玩家必须使用完全相同版本的 `STS2 LAN Connect`
- 建议直接对照 `mods/sts2_lan_connect/sts2_lan_connect.json` 中的版本号

### 需要回退到手动 LAN/IP

- 官方 Host / Join 页面里的手动 LAN 调试入口仍然保留，可作为排障回退方案
